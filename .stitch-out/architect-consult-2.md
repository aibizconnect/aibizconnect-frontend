ROOT CAUSES (all in the importer's grid handling, v1 limitations):

1. INNER GRID OUTSCORED THE LAYOUT GRID — findCardGrid scored candidates by card count (kids × 3 for flex/grid). In the About band (outer 2-col grid = [2x2 features grid | bio column]), the inner features grid scored 12 vs the outer layout grid's 6, so Bill picked the INNER grid and demoted the entire bio column (heading, quote, avatar, name) to loose "intro" text. FIX (D-173): the OUTERMOST flex/grid candidate now wins (depth asc, score desc tie-break); nested grids are handled by recursion (blocksFor, depth-capped at 3, flex/grid-only for nested passes). Non-flex fallback kept only at depth 0 for raw-HTML imports.

2. INTRO HARDCODED BEFORE THE GRID — band content outside the grid was always emitted ABOVE the grid row, regardless of DOM order. That's why the footer "© 2024..." line (which sits BELOW the 3 link columns in the design) jumped to the top of the footer. FIX: the walk now records skipIndex (out.length when it passes the skipped grid subtree), and the grid rows are spliced back at that exact position — before/after content preserved in DOM order.

3. COLUMNS = CARD COUNT, NOT GRID COLUMNS — a 2-col grid with 4 cards (sm:grid-cols-2 → 2x2) was emitted as columns:4 (one flat 4-up row), because our renderer draws exactly `columns` cells (extras are dropped, no wrapping). FIX: chunk the cards into one row per gridTemplateColumns count — 4 cards / 2 cols → two 2-col rows = a true 2x2. Wrapper-climbed style (D-171 navy card) only applied when the grid stays a single row (a per-chunk bg would draw the card background twice).

SUPPORTING GUARDS added: (a) grids inside <form> are never candidates (Stitch's 2-up name/email field wrapper would have been emitted as a row of loose label text duplicating the contact-form block); (b) cardish text test now uses cleanText (icon ligature words like "location_on" no longer make an icon+label flex pair look like a 2-card grid); (c) a lone grid row only merges into its band when their backgrounds AGREE — a navy rounded card inset in a white section stays nested so the white frame survives.
