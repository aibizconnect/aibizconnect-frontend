# AI DevOps Agent (Hybrid Mode)

This agent executes commands generated by Claude or other AIs.

## Modes:
- SAFE → auto-execute
- REVIEW → requires Ali's approval
- BLOCKED → never executed

## Logs:
- ai-agent-logs/commands.log
- ai-agent-logs/approvals.log
- ai-agent-logs/approval-queue.json

## Run manually:
ts-node ai-agent/agent.ts "npm run build"

## Daily Security Check

Run manually:
ts-node ai-agent/security-check.ts

This script verifies:
- Agent integrity
- Pending destructive actions
- Critical file presence
- Logs + queue health

Future versions will include:
- Supabase RLS checks
- Vercel env checks
- GitHub integrity checks
- Cloudflare DNS checks

## Approving Destructive Actions

To process pending REVIEW_REQUIRED commands:

ts-node ai-agent/approve.ts

You will be shown:
- The command
- The reason
- The impact
- Alternatives

Then you can approve or reject each one.

## Standalone HTTP Server

You can run the AI DevOps Agent as a standalone HTTP server:

ts-node ai-agent/server.ts

Default port: 4545  
Override with:

AI_AGENT_PORT=5555 ts-node ai-agent/server.ts

Endpoints:
- POST /run-command
  - Body: { "id"?: string, "command": string, "cwd"?: string, "meta"?: { reason, impact, alternatives } }
  - Response: { "success": boolean, "result"?: { ... }, "error"?: string }

- GET /health
  - Response: { "ok": true, "message": "AI Agent server is running." }

## Portable Config System

The agent uses:
- ai-agent/config/agent.config.json for non-secret config
- ai-agent/config/.env.vault for encrypted secrets (placeholder)
- loadConfig.ts and loadVault.ts to access values

To migrate the agent to a new machine:
1. Copy the entire ai-agent/ folder
2. Copy .env.vault
3. Install Node
4. Run: ts-node ai-agent/server.ts

## Encrypted Vault

The agent uses:
- ai-agent/config/.env.vault as an encrypted secret store
- VAULT_PASSWORD (env var) to decrypt it
- loadVault().get("KEY") to access values

Vault format:
- AES-256-GCM
- Base64-encoded blob
- Decrypted payload is a JSON object of key/value pairs

To rotate or update secrets:
1. Decrypt existing payload (if needed)
2. Edit the JSON
3. Re-encrypt with AES-256-GCM using VAULT_PASSWORD
4. Overwrite .env.vault with the new base64 blob
